# Abstract

Zero-configuration peer-to-peer (P2P) file sharing in local area networks typically sacrifices robust identity verification in favor of convenience, leaving users vulnerable to adversarial Man-In-The-Middle (MITM) and identity-spoofing attacks. Conversely, internet-reliant P2P systems inevitably introduce significant latency overhead, bypass local network efficiency, and depend upon centralized signaling servers or global trackers to mitigate trust. 

This paper introduces **WindWhisper**, an ephemeral, browser-native P2P file transfer framework built upon a custom application-layer transport protocol (ZEPHYR-1). By strictly decoupling signaling from data storage, the architecture utilizes a stateless Node.js WebSocket reflector alongside local Multicast DNS (mDNS) to discover neighboring peers without any network configuration or client installation. To secure the ephemeral handshake, WindWhisper employs a hybrid authentication model: users can either accept connections passively (for trusted environments) or enforce a 'Zero-Trust' locking layer utilizing Time-Based One-Time Passwords (TOTP) as a visual, out-of-band (OOB) side-channel. 

Furthermore, empirical testing of the implementation demonstrates that completely abandoning the black-box WebRTC data channel in favor of custom application-layer engineering yields highly verifiable results. By shifting the entire cryptographic burden—including Eliptic-Curve Diffie Hellman (ECDH P-256) session key derivation, AES-256-GCM binary chunk encryption, and a custom sliding-window Automatic Repeat reQuest (ARQ)—directly into the JavaScript WebCrypto API, the system achieves maximum bandwidth utilization while guaranteeing zero disk-spooling. The result is a highly robust, hardware-agnostic utility offering the unparalleled speed of local network discovery combined with the cryptographic integrity required for zero-trust environments.
